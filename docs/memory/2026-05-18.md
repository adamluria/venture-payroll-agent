## Decisions Made

- **`src/data/dataSource.js` as the single data access point** [Tier 1] — every view calls it; mock vs. live is one `useMock` boolean. When Sessions 3/4/5 wire up Salesforce/CoAdvantage/Five9, only this file needs editing. Mock mode stays forever as a demo fallback per the project rules.
- **Mock data shapes mirror planning.md exactly** [Tier 1] — Agent, weekly verification row, CommissionCalculation, Contest. The live data layer can return the same shapes from real APIs without touching the views.
- **Component split: primitives (`src/components/`) vs. views (`src/views/`)** [Tier 1] — components are data-agnostic; views know about specific data shapes. Keeps reuse honest.
- **Semantic color tokens** [Tier 1] — `flagged`/`approved`/`pending` instead of `red`/`green`/`amber`. Components reference the semantic name so the rule "flagged in coral" can never drift.
- **25 mock agents instead of 8** — matches the real headcount described in planning.md. Better for catching pagination/scrolling edge cases now rather than later.

## Bugs Fixed

- **`apartments` typo in original mock data** — the monolithic `app.jsx` had `apartments: 42` instead of `appointments: 42` for one agent, which meant their commission would calculate incorrectly when summed against the planning rules. Now fixed: every commission row is derived from a single `completed_appointments` value via `tierOf()` and the confirmed rate table.
- **Dangling `showContestConfig` state** — the Contest Rules button toggled state with no UI behind it. Now opens `ContestConfigModal` which displays current contests and a placeholder for the future config form.
- **Missing Google Fonts** — `index.html` referenced 'Outfit' and 'JetBrains Mono' via inline styles but never loaded the fonts. Added `<link rel="stylesheet">` to the Google Fonts CDN in `<head>`.
